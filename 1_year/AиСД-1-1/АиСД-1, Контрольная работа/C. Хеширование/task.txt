В хеш-таблице n ячеек, пронумерованных от 0 до n - 1 включительно. В таблице хранятся целые числа и используется разрешение коллизий при помощи цепочек. В таблице может храниться
            несколько одинаковых чисел. При добавлении числа x в таблицу, оно помещается в ячейку (x mod n), где x mod n — остаток от деления x на n. Чем меньше элементов в одной ячейке таблицы, тем быстрее поиск.
         Определим эффективность хеширования следующим образом:



где ni — число элементов в i-й ячейке. 
      
Вычислите эффективность хеширования (в процентах) для заданной хеш-таблицы.

Формат ввода

В первой строке содержится целое число n (2 ≤ n ≤ 100) — количество ячеек в хеш-таблице. Вторая строка содержит n целых чисел xi через пробел — это числа, хранящиеся в хеш-таблице (0 ≤ xi ≤ 109).
         
Формат вывода

Выведите единственное число — эффективность хеш-таблицы в процентах с точностью не менее двух знаков после запятой.
